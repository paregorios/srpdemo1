About:

Welcome the Syriac Reference Portal (syriaca.org) demo!

This demo is a working protype for linking digital manuscript repositories.

This demo was created by Tom Elliot and Hugh Cayless of New York University’s Institute for the Study of the Ancient World and by David Michelson, director of Syriaca.org at Vanderbilt University. Funding from the project was provided by The Mellon Foundation, The National Endowment for the Humanities, and The Balzan Foundation.

Objectives:

The main purpose of Syriaca.org is to provide Syriac scholars with tools searching across multiple online resources.

This demo accomplishes two objectives toward that purpose:

  1. Simultaneous cross-searching of online manuscript catalogues
  2. Helping Syriac scholars to identify what portions of large repositories like Fihrist or HMML are most relevant to their research.

Need:

Currently a scholar looking for manuscripts with a work by a particular Syriac author must search in several online manuscript catalogues each of which use different authority files and have different data structures.

This demo project offers a solution by showing how multiple searches could be combined into one search. For proof-of-concept, this demo combines results for three significant websites: Fihrist (in the UK), HMML (in the US), and ektobe (part of e-corpus, in France) into one dynamically generated results page.

The demo also accomplishes a second objective to help make projects like Fihrist (whose primary audience is scholars workling in Arabic) accessible for use by scholars (i.e. Syriac scholars) with different but related research needs who might not otherwise consult the information in Fihrist.

User Guide:

The current demo is organized around the author as the focus for discovery. The full site, however, will allow users to search based on other criteria (see the non-working tabs for titles of works, artifacts, and places)

Users begin with a list of authors which they could browse or search. Hovering over a link pulls up a brief window to help the user disambiguate their search.

Clicking on the link takes the user to the full authority file for the author (to help with disambiguation) and also dynamically generates links to manuscripts held by the partner projects.

From the list of manuscript results, the user can follow links taking them to the partner project site to view the manuscript contents.

Technical Explanation:

The demo uses XML data structure and the principles of Linked Open Data to link the partner sites together.

The authority file is an XML document composed in the EAC “Encoding Archival Context” schema. This authority file relies on URI’s (identifier numbers) from the Virtual International Authority File (VIAF).

The list of manuscripts is created by a javascript query of an RDF triple store created from the TEI or EAD records of the individual partner projects. That query returns a list of manuscripts and works for the matched VIAF ids, which the javascript writes it into the page as HTML.

Future Collaboration:

Syriaca.org is actively welcoming collaborators and project partners. Only a few steps are needed to make partner websites discoverable through the Syriaca.org.

  1. Each partner is invited to collaborate with Syriaca.org in the creation of the author or person authority file. Ideally we would work together to create and submit new names to the Virtual International Authority File. Each partner site would also incorporate these URIs into their data.
  2. The second step is to make arrangements for publishing in XML and RDF (for example from EAD or TEI records). Ideally, the RDF we use to query against would be extracted and published by the partner projects. This would be the best practice to keep the data current and would also benefit the partner projects in that they could allow other projects (beyond the Syriac Reference Portal) access to their records as well.

All prospective partner projects are invited to contact the project director, David Michelson.

Now for the actual demonstration:

This demo is organized around the author as the focus for discovery. The full site, however, will allow users to search based on other criteria (see the non-working tabs for titles of works, artifacts, and places)

We begin with a list of authors which users could browse or search.

Hovering over a link pulls up a brief window to help the user disambiguate their search.

Clicking on the link takes the user to the full authority file for the author (to help with disambiguation) and also dynamically generates links to manuscripts held by the partner projects.

For the technically curious, behind the the authority file is an XML document composed in the EAC “Encoding Archival Context” schema. This authority file relies on URI’s (identifier numbers) from the Virtual International Authority File (VIAF).

The result that you will see is actually generated live when the page loads, this means that each partner site can update their data without adversely affecting the results. This approach is made possible by the XML data structure and the principles of Linked Open Data. (In this regard, the Fihrist project serves as an example of best practices, very minimal changes were made to the data from Fihrist in order to pull that data live into the demo site).

Behind the list of manuscripts which loads, a bit of javascript sends a query to an RDF triple store created from the TEI or EAD records of the individual partner projects. That query returns a list of manuscripts and works for the matched VIAF ids, and the javascript writes it into the page as HTML.

You have now seen the basic working of the demo.
